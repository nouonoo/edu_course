## Назначение и цели создания системы

### Назначение

Платформа предназначена для автоматизации сбора, анализа и визуализации статистики прохождения корпоративных адаптивных курсов. Система используется для отслеживания прогресса сотрудников, оценки эффективности обучения и формирования отчётов для руководства.

### Цели создания системы

1. Отображение статистики в виде таблиц (рейтингов).
2. Формирование отчётов по успеваемости (по сотрудникам, по курсам).
3. Предоставление эксперту инструмента для назначения курсов и анализа данных.
4. Обеспечение обучающимся доступа к назначенным курсам и личному кабинету.
5. Предоставление администратору средств управления пользователями и каталогом курсов.

## Требования к системе

### Требования к системе в целом

1. Клиент-серверная архитектура: статический frontend (порт 8000) + backend Flask (порт 5000).
2. СУБД: Microsoft SQL Server, база `LearningPlatformDB`.
3. Интерфейс на русском языке, интуитивно понятный.
4. Разделы интерфейса по ролям:
   - **обучающийся:** «Моё обучение», «Рейтинг», профиль;
   - **эксперт:** «Назначения», «Рейтинг», «Моё обучение»;
   - **администратор:** «Пользователи», «Курсы».
5. Аутентификация по JWT (срок сессии 12 часов).
6. Поддержка нативных HTML-курсов и SCORM 2004.

### Требования к функциям (задачам), выполняемым системой

| № | Функция | Роль |
|---|---------|------|
| 1 | Авторизация и разграничение доступа | Все |
| 2 | Просмотр общей статистики (рейтинг) | Все |
| 3 | Фильтрация данных по сотруднику и курсу | Эксперт, админ |
| 4 | Назначение курсов сотрудникам | Эксперт |
| 5 | Прохождение курсов с сохранением прогресса | Обучающийся, эксперт |
| 6 | Формирование и выгрузка отчёта в Excel | Эксперт, админ |
| 7 | Управление пользователями (CRUD) | Администратор |
| 8 | Загрузка, редактирование, скачивание и удаление курсов | Администратор |
| 9 | Личный кабинет, загрузка фото | Обучающийся, эксперт (просмотр) |

### Требования к курсу «Безопасность»

- 6 оцениваемых разделов, порог прохождения 70%;
- последовательная навигация с gating интерактивов;
- кнопка «Закрыть курс» с сохранением прогресса;
- адаптивная вёрстка (ПК, планшет, мобильный).

## Порядок контроля и приёмки системы

### Виды, состав и методы испытаний

**Состав:** тестирование компонентов системы (веб-интерфейс, API, база данных, курс-плеер).

**Методы:** проверка функциональности по тест-кейсам (см. `docs/Тест-кейсы.md`).

**Критерии приёмки:**

- все тесты приоритета P1 пройдены на 100%;
- тесты P2 — не менее 95%;
- критические дефекты (блокирующие обучение) — 0.

### Требования к подготовке объекта автоматизации

1. Рабочее место с Windows 10 или выше и браузером Chrome / Edge / Firefox.
2. Подключение к локальной сети с сервером SQL Server.
3. Установленные Python 3.10+, ODBC Driver 17 for SQL Server.
4. Сервер (или локальная машина) с SQL Server и развёрнутой БД `LearningPlatformDB`.

### Организационно-штатные мероприятия

- назначение экспертов — конечных пользователей модуля назначений и отчётов;
- назначение администратора — управление пользователями и курсами;
- назначение технического специалиста для мониторинга сервера и БД.
