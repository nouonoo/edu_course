## Рисунок 5. Диаграмма последовательностей — прохождение курса

**Сценарий:** обучающийся запускает нативный курс, проходит раздел и возвращается в LMS.

```mermaid
sequenceDiagram
    actor Student as Обучающийся
    participant Browser as Браузер (frontend)
    participant Player as Курс-плеер (course-app.js)
    participant Bridge as LMSBridge
    participant API as Flask API
    participant DB as SQL Server

    Student->>Browser: Открыть instruction.html
    Browser->>API: GET /api/my-courses (JWT)
    API->>DB: SELECT Assignments, Courses
    DB-->>API: Список курсов
    API-->>Browser: JSON с assignment_id

    Student->>Browser: Нажать «НАЧАТЬ КУРС»
    Browser->>Player: Переход /course-content/2/index.html?assignment_id&authToken

    Player->>Bridge: LMSBridge.init()
    Bridge->>API: GET /api/courses/2?assignment_id=
    API->>DB: manifest + Section_progress
    DB-->>API: completedSections, scores
    API-->>Bridge: JSON manifest + progress
    Bridge-->>Player: Данные для resume

    Student->>Player: Пройти интерактивы раздела
    Student->>Player: Нажать «Далее»
    Player->>Bridge: completeSection(section_id, payload)
    Bridge->>API: POST /api/courses/2/sections/section-1/complete
    API->>DB: INSERT/UPDATE Section_progress
    DB-->>API: OK
    API-->>Bridge: updated progress
    Bridge-->>Player: Обновить UI

    alt Завершение всего курса
        Student->>Player: Экран «Заключение» → Завершить
        Player->>Bridge: finishCourse()
        Bridge->>API: POST /api/courses/2/finish
        API->>DB: UPDATE Assignments, User_result
        DB-->>API: status passed/failed
        API-->>Bridge: result
        Bridge-->>Browser: redirect returnUrl
    else Выход без завершения
        Student->>Player: «Закрыть курс» → Подтвердить
        Player->>Bridge: exitCourse()
        Bridge-->>Browser: redirect returnUrl
    end
```

## Рисунок 6. Диаграмма последовательностей — назначение курса

**Сценарий:** эксперт назначает курс сотруднику.

```mermaid
sequenceDiagram
    actor Expert as Эксперт
    participant UI as assignments.html
    participant API as Flask API
    participant DB as SQL Server
    actor Student as Обучающийся

    Expert->>UI: «Назначить курс»
    Expert->>UI: Заполнить форму (сотрудник, даты)
    UI->>API: POST /api/assignments
    API->>DB: Проверка дубликата active
    alt Дубликат есть
        DB-->>API: Ошибка
        API-->>UI: 400 + сообщение
    else Успех
        API->>DB: INSERT Assignments
        DB-->>API: assignment_id
        API-->>UI: 200 OK
        UI-->>Expert: Уведомление об успехе
        Student->>UI: instruction.html
        UI->>API: GET /api/my-courses
        API->>DB: SELECT с новым назначением
        API-->>UI: Курс доступен, «НАЧАТЬ КУРС»
    end
```

## Рисунок 7. Диаграмма последовательностей — формирование отчёта

```mermaid
sequenceDiagram
    actor Expert as Эксперт
    participant UI as rating.html
    participant API as Flask API
    participant DB as SQL Server

    Expert->>UI: Нажать «Отчёт»
    UI->>Expert: Модальное окно фильтров
    Expert->>UI: Подтвердить
    UI->>API: GET /api/report?user_id&course_id&...
    API->>DB: SELECT Users, Courses, User_result, Assignments
    DB-->>API: Строки данных
    API->>API: create_excel_report()
    API-->>UI: course_report.xlsx (binary)
    UI-->>Expert: Скачивание файла
```

## Описание участников

| Участник | Назначение |
|----------|------------|
| Обучающийся / Эксперт | Конечный пользователь |
| Браузер (frontend) | HTML/JS страницы LMS на порту 8000 |
| Курс-плеер | index.html + course-app.js + course-interactions.js |
| LMSBridge | Мост между плеером и REST API |
| Flask API | backend/app.py на порту 5000 |
| SQL Server | Хранение Users, Assignments, Section_progress, User_result |
