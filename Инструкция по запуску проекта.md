# Инструкция по запуску проекта

Платформа электронного обучения (LMS) для прохождения курсов, назначения обучения сотрудникам, ведения рейтинга и загрузки SCORM-пакетов.

**Корневая папка проекта:**

```
C:\Users\User\OneDrive\Рабочий стол\курсовая\kursovaya
```

---

## 1. Состав проекта

| Папка / файл | Назначение |
|---|---|
| `backend/` | Сервер Flask (API, раздача курсов, SCORM, работа с БД) |
| `frontend/` | Статические HTML/JS-страницы интерфейса LMS |
| `backend/courses/` | Файлы курсов (нативные и SCORM) |
| `backend/courses/bezopasnost/` | Курс «Безопасность» (нативный электронный курс) |
| `backend/dump.sql` | Создание БД и начальные данные |
| `backend/migration.sql` | Дополнительные таблицы и миграции |
| `backend/config.py` | Строка подключения к SQL Server и пути |
| `картинки из курса/` | Исходные SVG-изображения для сборки курса |
| `.venv/` | Виртуальное окружение Python (создаётся при установке) |

### Архитектура

- **Frontend** — статический сайт на порту **8000** (`http://127.0.0.1:8000`)
- **Backend** — Flask API на порту **5000** (`http://127.0.0.1:5000`)
- **База данных** — Microsoft SQL Server (`LearningPlatformDB`)
- Курсы открываются через backend: `/course-content/<course_id>/...`

> Frontend и backend должны работать **одновременно**. Адреса API зашиты в `frontend/common.js` (`http://127.0.0.1:5000`).

## 2. Требования к окружению

### Обязательно

1. **Windows 10/11**
2. **Python 3.10+** — скачать с [python.org](https://www.python.org/downloads/), при установке включить **«Add Python to PATH»**
3. **Microsoft SQL Server** (Express / Developer / LocalDB)
4. **ODBC Driver 17 for SQL Server** — [скачать у Microsoft](https://learn.microsoft.com/sql/connect/odbc/download-odbc-driver-for-sql-server)
5. **SSMS** (SQL Server Management Studio) — для выполнения SQL-скриптов (рекомендуется)

### Проверка Python

В PowerShell:

```powershell
py -3 --version
```

> На Windows команда `python` часто открывает заглушку Microsoft Store и **не запускает** проект. Используйте **`py -3`**.

## 3. Установка базы данных

### 3.1. Создание БД

1. Запустите SQL Server.
2. Откройте SSMS, подключитесь к серверу (обычно `localhost` или `(localdb)\MSSQLLocalDB`).
3. Выполните скрипт:

```
backend\dump.sql
```

Скрипт создаёт базу `LearningPlatformDB`, таблицы и тестовых пользователей.

### 3.2. Миграции

После `dump.sql` выполните:

```
backend\migration.sql
```

Скрипт добавляет таблицы `Assignments`, `Section_progress` и связанные поля.

> При **первом запуске backend** также вызывается `ensure_schema()` — недостающие таблицы и столбцы создаются автоматически. Но начальные данные из `dump.sql` всё равно нужны.

### 3.3. Настройка подключения

Откройте `backend/config.py` и при необходимости измените строку:

```python
DB_CONNECTION_STRING = (
    "DRIVER={ODBC Driver 17 for SQL Server};"
    "SERVER=localhost;"
    "DATABASE=LearningPlatformDB;"
    "Trusted_Connection=yes;"
    "TrustServerCertificate=yes;"
)
```

| Параметр | Когда менять |
|---|---|
| `SERVER` | Если SQL Server не на `localhost` (имя экземпляра, `.\SQLEXPRESS` и т.д.) |
| `DATABASE` | Если база названа иначе |
| `Trusted_Connection=yes` | Windows-аутентификация. Для SQL-логина замените на `UID=...;PWD=...` |

### 3.4. Проверка подключения

```powershell

py -3 test_db.py
```

Ожидаемый результат: `УСПЕХ! Подключение установлено.`

## 8. Курсы в системе

### 8.1. Нативный курс «Безопасность»

- **ID в БД:** `course_id = 2`
- **Папка:** `backend/courses/bezopasnost/`
- **Манифест:** `backend/courses/bezopasnost/course.json`
- **Точка входа:** `index.html`
- **Тип:** `native` (электронный курс)

Структура курса:

| № | Раздел | Оценивается в рейтинге |
|---|---|---|
| — | Обложка (сплеш) | нет |
| 1 | Инструкция по навигации | нет |
| 2 | Введение | нет |
| 3–8 | Основные разделы (section-1 … section-6) | да (6 разделов, до 100%) |
| 9 | Заключение | нет |

Особенности:

- Закрытая последовательная навигация
- «Далее» блокируется, пока не просмотрены интерактивы слайда
- На тестах и практике блокируется меню и навигация
- Попытки на задания не ограничены до правильного ответа
- Кнопка **«Закрыть курс»** — выход в LMS с сохранением пройденных разделов
- Адаптивная вёрстка (ПК, планшет, мобильный)

Исходные изображения: `картинки из курса/`.

### 8.2. SCORM-курсы

- Загружаются администратором на `admin-courses.html` (ZIP с `imsmanifest.xml`)
- Хранятся в `backend/courses/<папка>/`
- Запускаются через `player.html` и SCORM 2004 API

## 11. Типичные проблемы

### `python` не запускается / выводит только «Python»

Используйте `py -3` вместо `python`.

### Ошибка подключения к БД (`pyodbc`)

1. SQL Server запущен (служба **SQL Server (MSSQLSERVER)** или **SQLEXPRESS**).
2. Установлен **ODBC Driver 17**.
3. В `config.py` верный `SERVER`.
4. База `LearningPlatformDB` создана (`dump.sql`).

### «Курс недоступен» / «Требуется авторизация»

Курс открывается только через LMS с параметрами `assignment_id` и `authToken`. Прямое открытие `index.html` курса не работает — войдите в LMS и нажмите **«НАЧАТЬ КУРС»**.

### «Курсы не назначены» / «Не удалось создать назначение»

1. Выполните `migration.sql` (таблица `Assignments`).
2. Перезапустите backend.
3. Назначьте курс экспертом на `assignments.html`.

### «Файлы курса не найдены»

1. Папка `backend/courses/bezopasnost/` существует.
2. В БД у курса `storage = 'bezopasnost'` (обновляется при старте backend и в `migration.sql`).

### CORS / сетевые ошибки в браузере

1. Backend на `http://127.0.0.1:5000`.
2. Frontend на `http://127.0.0.1:8000`.
3. Оба сервера запущены.
4. В `frontend/common.js` не меняли `API_URL` без соответствующих правок backend.

### Курс отображается некорректно

Для нативного курса масштаб браузера — **100%** (указано в инструкции по навигации курса).
